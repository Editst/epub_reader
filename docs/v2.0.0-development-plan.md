# EPUB Reader v2.0.0 开发计划（TDD）

> 基线来源：`docs/comprehensive_repost.md` 审计项 D-2026-09 / D-2026-10 与 `docs/ROADMAP.md` v2.0.0 里程碑。  
> 计划版本：v2.0.0  
> 制定日期：2026-03-12

---

## 1. 目标与范围

### 1.1 版本目标
- 提升 ETA 估算可信度：从“等权平均”升级为“会话加权 + 跳读识别 + 低样本提示”。
- 降低 `locations.generate()` 对首屏与交互的阻塞影响：引入 idle 调度包装与可见进度反馈。
- 优化首页书架加载感知性能：支持骨架屏 + 渐进插入（每本就绪即渲染）。

### 1.2 对应债务闭环
- D-2026-09（阅读速度模型）
- D-2026-10（locations 主线程长任务）
- ROADMAP v2.0.0 P-1 / P-2 / P-3

### 1.3 非目标（本版本不做）
- `reader.js` 大规模拆分（留给 v2.1.0 R-1/R-2/R-3）。
- TOC 虚拟滚动完整实现（先补调度与进度机制，虚拟滚动保留后续迭代）。
- Annotations 算法专项（v2.3.0/v2.4.0）。

---

## 2. TDD 开发策略

1. **先计划**：明确需求分解、验收标准、回滚策略。
2. **先测试**：
   - 纯函数单测（ETA 估算与会话权重）。
   - 静态契约测试（reader/home 是否引入目标机制）。
   - 回归测试（现有 v1.9.3 套件不退化）。
3. **再实现**：最小改动实现功能，保证可维护性。
4. **再修复**：根据测试失败回补实现细节。
5. **文档与版本同步**：`ROADMAP/CHANGELOG/README/manifest` 全部一致。

## 3. 详细任务拆分

## 3.1 P-1 阅读速度/ETA 模型升级

### 设计要点
- 新增工具函数：
  - `Utils.computeSessionWeight(deltaProgress, deltaSeconds)`
  - `Utils.estimateRemainingMinutes(options)`
- 对 session 样本应用加权：
  - 连续阅读：权重 1.0
  - 轻度跳读：权重 0.6
  - 明显跳跃：权重 0.2
- 低样本提示策略：
  - 历史与当前会话均不足阈值时，ETA 文案显示“估算中”。

### 验收标准
- 中途开书场景 ETA 不再长时间显示异常尖峰。
- 低样本时不展示“看似精确但不可信”的分钟数。
- 相关函数具备独立单元测试。

---

## 3.2 P-2 locations 生成调度与进度反馈

### 设计要点
- 新增 `scheduleLocationsGeneration` 调度器：
  - 优先 `requestIdleCallback`。
  - 无 rIC 时回退 `setTimeout`。
- `reader.js` 中将 `book.locations.generate(1600)` 放入调度器。
- 通过 `showLoading(true, '...')` 按阶段反馈：
  - “准备定位索引...”、“生成阅读定位索引...”、“定位索引就绪”。

### 验收标准
- 代码中不再直接在主流程同步触发 generate。
- 具备 rIC / fallback 双路径测试。

---

## 3.3 P-3 书架流式渲染 + 骨架屏

### 设计要点
- 首屏先渲染固定数量骨架卡片（默认 6）。
- 并发拉取每本书封面与元数据，**每本完成即 append**。
- 全部渲染完成后移除 loading 状态。

### 验收标准
- 不再“等所有 Promise 完成后一次性渲染”。
- `home.css` 提供骨架动画样式。
- `home.js` 含流式渲染入口函数。

## 6. 风险与回滚

- 风险1：ETA 权重参数过激导致估算震荡。  
  - 缓解：参数集中在 `Utils`，可快速调参。
- 风险2：locations 进度反馈误导用户。  
  - 缓解：文案明确“生成中”，完成后立即切换。
- 风险3：骨架屏导致样式回归。  
  - 缓解：新增静态测试 + 手动截图验证。

回滚策略：
- 逻辑层（Utils/reader/home）均可按文件回滚，不影响存储 schema。
